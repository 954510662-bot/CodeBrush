## 👋 首次贡献者指南

第一次为开源项目做贡献？别担心！我们为你准备了专门的任务。

### 寻找适合你的任务

查看我们的 Issue 标签：

| 标签 | 说明 | 预期时间 |
|------|------|----------|
| `good first issue` | 专门为新人准备的简单任务 | 30分钟内 |
| `help wanted` | 需要帮助的中等难度任务 | 2-4小时 |
| `documentation` | 文档相关任务 | 1-2小时 |
| `design needed` | 需要设计贡献 | 灵活 |

### 如何提交你的第一个 PR

1. 选择一个 `good first issue` 任务
2. Fork 这个仓库
3. 创建一个新分支
4. 进行修改
5. 提交你的代码
6. 创建 Pull Request

详细步骤请参考下方的 [贡献流程](#贡献流程)。

## 🛠️ 开发环境设置

### 前置要求
- Node.js >= 20.x
- pnpm >= 8.x
- Git

### 克隆仓库
```bash
git clone https://github.com/954510662-bot/CodeBrush.git
cd CodeBrush
```

### 安装依赖
```bash
pnpm install
```

### 启动开发服务器
```bash
pnpm run dev
```

### 构建生产版本
```bash
pnpm run build
```

### 运行测试
```bash
pnpm run test
```

## 📖 贡献流程

### 1. 选择或创建 Issue
- 在 [Issues](https://github.com/954510662-bot/CodeBrush/issues) 页面找到适合你的任务
- 如果没有找到合适的，你可以创建一个新 Issue 来讨论你的想法

### 2. 讨论和规划
- 在 Issue 中与维护者讨论实现方案
- 确保你的工作不会与其他人重复

### 3. Fork 仓库
点击仓库页面右上角的 "Fork" 按钮

### 4. 创建分支
```bash
git checkout -b feature/你的功能名
# 或者
git checkout -b fix/你的修复名
```

### 5. 进行修改
- 编写你的代码
- 确保代码遵循项目的风格指南
- 添加必要的测试

### 6. 提交修改
```bash
git add .
git commit -m "feat: 添加新功能"  # 遵循提交信息规范
```

### 7. 推送到 Fork
```bash
git push origin feature/你的功能名
```

### 8. 创建 Pull Request
- 前往你的 Fork 仓库
- 点击 "Pull request" 按钮
- 填写 PR 模板中的所有信息
- 提交 PR

## 📝 代码风格指南

### 基本准则
- 使用 TypeScript 类型
- 保持代码简洁清晰
- 添加必要的注释
- 遵循现有的代码风格

### 前端代码规范
- 使用 2 空格缩进
- 使用单引号
- 使用尾分号
- 组件命名使用 PascalCase
- 函数命名使用 camelCase

### Git 工作流
- 保持提交原子化
- 一个 PR 只解决一个问题
- 确保分支是最新的

## 📌 提交信息规范

我们使用 Conventional Commits 规范：

### 格式
```
<类型>(<范围>): <描述>

<正文>

<页脚>
```

### 类型
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关

### 示例
```
feat: 添加撤销/重做功能

- 实现状态历史管理
- 添加快捷键支持

Closes #123
```

## 🚀 PR 指南

### PR 标题
使用与提交信息相同的格式

### PR 描述模板
我们会提供模板，请务必：
- 清晰描述改动内容
- 关联相关 Issue（如 Closes #123）
- 添加截图或演示（如适用）
- 检查所有复选框

### PR 大小
- 尽量保持 PR 小巧聚焦
- 大功能建议拆分成多个小 PR

### 代码审查
- 维护者会在 24-48 小时内响应
- 请积极参与讨论和修改
- 代码审核通过后会合并
